# 뉴스 클리핑 자동화 서비스 PRD

**작성일**: 2025-12-08  
**버전**: 1.0  
**작성자**: 레고  
**프로젝트명**: Political News Clipper (가칭)

---

## 1. 개요

### 1.1 목적
정당활동에 필요한 뉴스를 자동으로 수집, 분석, 요약하여 매일 노션 페이지로 제공하는 자동화 서비스

### 1.2 배경
- 정의당 경상남도당 활동
- 매일 다량의 관련 뉴스를 확인해야 하는 상황
- 수동으로 뉴스를 찾고 읽는 시간 소모가 큼
- 중요한 뉴스를 놓칠 위험 존재

### 1.3 목표
- 일일 뉴스 모니터링 시간을 50% 이상 단축
- 중요 뉴스 놓침 방지율 90% 이상
- 노션에 자동으로 정리된 형태로 제공하여 즉시 활용 가능하도록 구성

## 2. 핵심 기능 (Core Features)

### 2.1 뉴스 수집 (News Collection)
**설명**: 구글 알리미를 통해 지정된 키워드의 뉴스를 자동 수집

**주요 키워드 카테고리**:
- 일반: "경남도당", "정의당", "경남", "경상남도"
- 주요 이슈: "노동", "산재", "사망", "여성", "성평등",  "생태", "환경", "기후
", "동물복지"
- 선거: "2026 지방선거", "지방선거"

**주요 언론사**
- 경남도민일보 https://www.idomin.com/
- 경남신문 https://m.knnews.co.kr/index.html
- kbs창원 https://changwon.kbs.co.kr/
- mbc경남 https://mbcgn.kr/01_new/new01.asp

**기술 구현**:
- 구글 알리미 RSS 피드 활용
- 또는 Google News API / News API 활용
- 수집 주기: 매일 오전 7시, 오후 6시 (2회)

### 2.2 AI 기반 뉴스 선별 (AI Filtering)
**설명**: 수집된 뉴스 중 실제로 중요하고 관련성 높은 뉴스만 선별

**선별 기준**:
1. **관련성 점수** (Relevance Score)
   - 키워드와의 직접적 연관성
   - 정치활동에 미치는 영향도
   - 지역 연관성

2. **중요도 점수** (Importance Score)
   - 뉴스의 파급력
   - 즉각적인 대응 필요 여부
   - 언론사 신뢰도

3. **필터링 기준**
   - 중복 뉴스 제거
   - 광고성 콘텐츠 제거
   - 관련성 낮은 뉴스 제거 (threshold: 60점 이상만 통과)

**AI 모델**:
- Claude API (Anthropic) 또는 GPT-4 활용
- Prompt Engineering으로 선별 기준 구체화

### 2.3 뉴스 요약 (Summarization)
**설명**: 선별된 뉴스를 간단명료하게 요약

**요약 형식**:
```markdown
### [뉴스 제목]
- **언론사**: OO일보
- **발행일시**: 2025-12-08 14:30
- **주요 키워드**: #LNG발전소 #기후위기 #경남
- **중요도**: ⭐⭐⭐⭐ (5점 만점)
- **한줄요약**: 핵심 내용을 한 문장으로
- **상세요약**: 
  - 배경: 무슨 일이 있었는지
  - 현황: 현재 상황
  - 영향: 우리 활동에 미치는 영향
  - 대응: 필요한 액션 아이템
- **원문링크**: [기사 보기](URL)
```

**요약 품질 기준**:
- 객관적 사실 중심
- 정치적 편향 최소화 (단, 진보 정치인 입장에서 중요한 관점 포함)
- 액션 아이템 명확히 제시

### 2.4 노션 자동 생성 (Notion Integration)
**설명**: 매일 자동으로 노션 페이지를 생성하고 분석된 뉴스를 정리

**노션 페이지 구조**:
```
📰 [날짜] 일일 뉴스 클리핑
├── 📊 오늘의 요약
│   ├── 총 뉴스 수: XX건
│   ├── 주요 뉴스: X건
│   └── 긴급 대응 필요: X건
├── 🚨 긴급 뉴스 (중요도 5점)
├── ⭐ 주요 뉴스 (중요도 4점)
├── 📌 일반 뉴스 (중요도 3점)
└── 🔍 키워드별 분류
    ├── 정치
    ├── 기후/환경
    └── 지역 이슈
```

**노션 데이터베이스 속성**:
- 제목 (Title)
- 카테고리 (Select): 정치/기후/지역
- 중요도 (Number): 1-5점
- 키워드 (Multi-select)
- 언론사 (Text)
- 발행일시 (Date)
- 요약 (Text - Page content)
- 원문링크 (URL)
- 대응완료 (Checkbox)

## 3. 시스템 아키텍처

### 3.1 기술 스택
- **언어**: Python 3.10+
- **뉴스 수집**: 
  - feedparser (RSS)
  - requests (API 호출)
- **AI**: 
  - Anthropic Claude API (추천)
  - OpenAI GPT-4 (대안)
- **노션**: 
  - notion-client (Python SDK)
- **스케줄링**: 
  - APScheduler (Python)
  - 또는 Cron (Linux/Mac)
  - 또는 GitHub Actions (클라우드 자동화)
- **데이터 저장**: 
  - SQLite (로컬 캐시)
  - JSON (설정 파일)

### 3.2 시스템 플로우
```
[구글 알리미 RSS]
      ↓
[뉴스 수집 모듈]
      ↓
[중복 제거 & 캐싱]
      ↓
[AI 분석 모듈]
  ├→ 관련성 평가
  ├→ 중요도 평가
  └→ 요약 생성
      ↓
[노션 API 모듈]
  ├→ 일일 페이지 생성
  └→ 뉴스 항목 추가
      ↓
[로그 & 알림]
```

### 3.3 배포 방식
**Phase 1 (MVP)**: 로컬 실행
- 개인 PC/Mac에서 실행
- Cron으로 스케줄링

**Phase 2**: 클라우드 자동화
- GitHub Actions (무료)
- 또는 AWS Lambda + EventBridge
- 또는 Google Cloud Functions + Scheduler

## 9. FAQ

### Q1. 왜 구글 알리미를 쓰나요?
A. 무료이고, RSS 피드를 제공하여 자동화가 쉽습니다. 단, News API 등으로 교체 가능합니다.

### Q2. AI 모델은 어떤 걸 쓰나요?
A. Claude Sonnet 4.5를 추천합니다. 요약 품질이 좋고, 한국어도 잘 이해합니다. GPT-4도 대안으로 가능합니다.

### Q3. 하루에 몇 번 실행되나요?
A. MVP는 하루 1-2회, 이후 필요에 따라 조정 가능합니다.

### Q4. 노션 말고 다른 도구는 안되나요?
A. Google Sheets, Airtable, Notion 등 다양한 도구로 확장 가능합니다.

### Q5. 모바일에서도 볼 수 있나요?
A. 네, 노션 모바일 앱에서 바로 확인 가능합니다.
